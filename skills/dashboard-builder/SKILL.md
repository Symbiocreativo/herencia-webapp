# Dashboard Builder Skill

## Cuándo usar este skill
Usa este skill cuando tengas que crear o modificar páginas de dashboard o paneles de seguimiento dentro de `src/app/**`.

## Objetivo
Construir interfaces claras, estables y sobrias para esta app de herencia.

## Reglas
- trabajar solo sobre los archivos pedidos
- no modificar otras rutas sin indicación explícita
- usar TypeScript
- no agregar dependencias nuevas
- preferir datos mock fijos al inicio
- mantener diseño simple, limpio y profesional
- usar texto correcto en español cuando corresponda
- guardar archivos en UTF-8
- no entregar trabajo terminado si el archivo queda con errores de codificación, parseo o build

## Estructura esperada
En dashboards o paneles, priorizar:
- título claro
- subtítulo breve
- tarjetas o bloques legibles
- estado actual
- bloqueo principal
- próximo paso
- enlace a detalle

## Antes de terminar
Verificar:
- que el archivo editado se pueda leer bien
- que no haya caracteres corruptos
- que la ruta funcione
- que el cambio sea coherente con `AGENTS.md`
